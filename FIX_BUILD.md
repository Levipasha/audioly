# Fix Build Issues - Step by Step

## Problem
The build is failing because:
1. The `android` folder exists but native modules aren't properly configured
2. Dependency version mismatches with Expo SDK 54
3. `react-native-reanimated` is on v3 but needs v4

## Solution Steps

### Step 1: Remove the android folder (if it exists locally)
```bash
# On Windows (PowerShell)
Remove-Item -Recurse -Force android

# On Mac/Linux
rm -rf android
```

### Step 2: Update dependencies
```bash
# Install updated dependencies
npm install

# Or use Expo's dependency checker
npx expo install --fix
```

### Step 3: Clean and reinstall
```bash
# Remove node_modules and lock file
rm -rf node_modules
rm package-lock.json  # or yarn.lock if using yarn

# Reinstall
npm install
```

### Step 4: Verify with expo doctor
```bash
npx expo doctor
```

### Step 5: Build the AAB
```bash
eas build --platform android --profile production
```

## What Was Fixed

1. ✅ Updated `.gitignore` to exclude `/android` and `/ios` folders (using CNG/Prebuild)
2. ✅ Updated `react-native-reanimated` from `^3.16.6` to `~4.1.1`
3. ✅ Updated Expo packages to match SDK 54:
   - `expo`: `~54.0.30` → `~54.0.31`
   - `expo-constants`: `~18.0.12` → `~18.0.13`
   - `expo-notifications`: `~0.32.15` → `~0.32.16`
   - `@react-native-community/slider`: `^5.1.1` → `5.0.1`

## Important Notes

- The `android` folder will be auto-generated by EAS Build using Prebuild
- If you need to test locally, use `npx expo prebuild` to generate native folders
- Make sure to commit the updated `package.json` and `.gitignore` files
